# CW Schema: Commit Pattern (_autosave: 0)

## When to use
- Document has virtual fields (password, token, OTP, card number)
- Document requires explicit user action before any persistence
- All field input should accumulate until terminal signal

## Schema requirements

### `_autosave: 0`
Required. Blocks data branch from auto-persisting on field change.
All field changes accumulate in `run_doc.input` only.

### `virtual: 1` on fields
Fields marked virtual are:
- Never persisted to DB
- Accumulated in input until commit signal
- Read by sideEffect from target.data[0] after merge
- Stripped from target.data[0] before persist

### `commit: true` on terminal transition
Declared on the transition that triggers full pipeline:
- merge input → target.data[0]
- _preflight validates all reqd fields
- sideEffect reads target.data[0] (including virtual)
- strip virtual from target.data[0]
- persist target.data[0] (real fields only)

### `sideEffects` on terminal transition
Declared on same transition as `commit: true`.
SideEffect fires after merge and validate, before strip and persist.
SideEffect reads complete target.data[0] including virtual fields.

## Accepted limitations
- All input lost if browser closes before commit signal
- No partial save — all or nothing
- Multi-step navigation (dim1) does not persist intermediate steps
- Real fields accumulate same as virtual — lost before commit

## Example
```json
{
  "schema_name": "UserPublicProfile",
  "_autosave": 0,
  "fields": [
    { "fieldname": "email",     "virtual": 1, "reqd": 1 },
    { "fieldname": "password",  "virtual": 1, "reqd": 1 },
    { "fieldname": "full_name", "reqd": 1 }
  ],
  "_state": {
    "0": {
      "values": [0, 1],
      "options": ["Draft", "Submitted"],
      "transitions": { "0": [1] },
      "labels": { "0_1": "Sign Up" },
      "primary": { "0_1": true },
      "commit": { "0_1": true },
      "sideEffects": { "0_1": "Adapter.pocketbase.signUp" }
    }
  }
}
```

## Pipeline on commit signal
signal 0.0_1 fires
→ validate transition ✓
→ merge input → target.data[0]
→ _preflight validates all reqd
→ sideEffect reads target.data[0]
→ if signal = '-1' → stop
→ strip virtual from target.data[0]
→ persist target.data[0]

## Pipeline on navigation signals (dim1)
signal 1.x_y fires
→ validate transition ✓
→ update _state only
→ no merge, no strip, no persist
