# CW Framework — Claude Session Protocol

Paste at the start of every session alongside the preamble.

---

## BEFORE EVERY TASK

You provide:
Task:   [one sentence outcome — what should be true after, not how]
Files:  [files you expect to be touched — can be wrong, Claude corrects]
Verify: [how you will confirm it worked — IIFE, visual check, etc.]

Claude responds with declaration only — no code until you confirm:
WILL CHANGE:
[file] [line/function]: [exact change in one sentence]
WILL NOT TOUCH (noticed, leaving alone):
[thing]: [one sentence — why out of scope]
ASSUMPTIONS:
[anything unclear — stated as a question, not a conclusion]
VERIFY:
[exact IIFE or check — agreed before any code is written]

You respond with exactly one of:
- `proceed` — both lists correct, write the code
- `add [X] to WILL CHANGE` — promote from second list
- `remove [X] from WILL CHANGE` — Claude overreached
- `answer to assumption` — resolve before proceeding

No code until you say proceed or answer assumptions.

---

## DURING IMPLEMENTATION
SIMPLICITY   Minimum code that solves the problem.
No speculative features, abstractions, or flexibility not requested.
If 200 lines would do in 50 — rewrite before submitting.
STYLE        Match existing code exactly — naming, spacing, patterns.
Do not improve adjacent code, comments, or formatting.
SCOPE        Every changed line traces directly to WILL CHANGE list.
If a change requires touching something not in WILL CHANGE — stop, ask.

---

## AFTER IMPLEMENTATION

Claude states:
CHANGED:
[file] [line/function]: [what was done]
RUN VERIFY:
[exact IIFE or check to paste in console]

You run verify and respond with one of:
- result output — paste IIFE result, not description of it
- `pass` — verify confirmed

If verify fails: paste the IIFE output. Claude reads the actual error, not your summary of it.

---

## WHAT NEVER HAPPENS

- Code before `proceed`
- Changes not in WILL CHANGE list
- WILL NOT TOUCH list is empty when Claude noticed related issues
- Verify skipped or described instead of run
- Claude fixes things in WILL NOT TOUCH without being asked
