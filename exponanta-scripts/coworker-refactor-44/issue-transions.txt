The reason _state keeps history is audit trail and multi-dim independence:
dim 0: docstatus lifecycle    0→1→2→0  (submit/approve/reset)
dim 1: agent_status lifecycle 0→1→2→3  (pending/running/done/failed)
Each dim's signals are independent — clearing dim 0 signals would also wipe dim 1 signals if you clear by prefix naively.
And the history of "0.0_1": "1", "0.1_2": "1" tells you the document was submitted then approved — that's the audit trail. Wiping it loses that.
The real problem is _getDimValue doesn't handle multiple cycles of the same dim correctly. It sees 0.0_1: 1 and 0.2_0: 1 and gets confused about current state.
The correct fix — _getDimValue should find the terminal state by walking the signal chain, not just looking at any signal:
javascript// find the signal whose `to` value is not the `from` of any other signal
// that's the current state
const prefix = dim + '.'
const signals = Object.entries(state)
  .filter(([k, v]) => k.startsWith(prefix) && v === '1')
  .map(([k]) => {
    const parts = k.slice(prefix.length).split('_')
    return { from: parseInt(parts[0]), to: parseInt(parts[1]) }
  })

// to values that are never a from = terminal state
const froms = new Set(signals.map(s => s.from))
const terminal = signals.find(s => !froms.has(s.to))
return terminal ? terminal.to : dimDef?.values?.[0] ?? 0
This correctly resolves current state regardless of how many cycles have occurred. History preserved, current state unambiguous.
