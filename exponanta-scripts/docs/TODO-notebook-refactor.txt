A Script cell is just a Run doc where target_doctype = 'Script'. That's the only difference. The "uniqueness" is purely in how runChain handles it — compile code into fn() instead of calling CW.run().
Everything else is identical — it has a name, lives in PocketBase, is discoverable in the Run list, has _changes, can be referenced by multiple notebooks.
There is no separate Script concept needed at the cell level.
