7:40 PM
Yes. htm — a tiny library that gives you tagged template literals, no build step needed:

html
<script src="https://unpkg.com/htm/preact/standalone.module.js"></script>
Or with React UMD:

javascript
const html = htm.bind(React.createElement)
Then instead of:

javascript
ce('section', { className: 'mb-5' },
  ce('h2', { className: 'mb-4' }, 'Upcoming Events'),
  ce('div', { className: 'container' })
)
You write:

javascript
html`
  <section class="mb-5">
    <h2 class="mb-4">Upcoming Events</h2>
    <div class="container"></div>
  </section>
`
Drop-in for your stack
javascript
// after React UMD loads
const html = htm.bind(React.createElement)

const EventCard = ({ doc }) => html`
  <div class="card event-card evh evh--v1 mb-3">
    <div class="evh__img">
      <img src=${asset(doc.slug, 'hero-image.jpg')} alt=${doc.title} />
    </div>
    <div class="evh__body">
      <h3 class="evh__title">${doc.title}</h3>
      <p class="evh__desc">${doc.deck}</p>
    </div>
  </div>
`
No build, no JSX transpiler, no Babel. Works with your React 18 UMD exactly as-is. The only difference from JSX is class instead of className.

is this close to ideal

7:43 PM
For your stack — yes, very close. Let me assess honestly:

What makes it close to ideal
No build step — stays true to your no-build constraint, loads from CDN
Readable — HTML tags instead of ce() nesting, maintainable by anyone
React-compatible — same reconciler, same hooks, same component model
Tiny — htm is ~700 bytes gzipped

The one real concern
Script load order — htm must load after React UMD but before any component file that uses html. In your strict load order contract this is one more dependency to manage.

What would make it fully ideal
If html is defined once globally:

javascript
// in CW-ui.js, after React loads
globalThis.html = htm.bind(React.createElement)
Then every component file just uses html directly — no import, no per-file setup. Consistent with how your stack already exposes React, ce, and components on globalThis.

The only alternative closer to ideal
Raw JSX with esbuild — but that requires a build step which you've deliberately avoided. So for your constraints, htm is the right answer.

One line in CW-ui.js, one CDN script tag, done.
