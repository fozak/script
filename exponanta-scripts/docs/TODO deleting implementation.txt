Claude responded: Frappe solves it with two separate concepts:
Frappe solves it with two separate concepts:

1. Cancel — docstatus = 2 — business record stays visible, immutable, shows "Cancelled" badge. Used for submitted documents that need audit trail — invoices, stock entries, salary slips. You can see it happened, you just can't edit it.

2. Delete — actual removal from DB — used for draft documents (docstatus = 0) that were never submitted. No audit trail needed. Also available to System Manager for cancelled docs in some cases.

The rule: if a document was ever submitted (docstatus = 1), you cancel it, not delete it. If it was never submitted (still draft), you delete it.

Your current system has delete handler setting docstatus = 2 — which is cancel, not delete. That's why UI gets polluted — cancelled records keep showing up.

Clean fix:

0_2 transition → actual delete from DB (draft never submitted, safe to remove)
1_2 transition → cancel (submitted, needs audit trail, stays in DB)
Add a filter in all list/grid selects: docstatus != 2 by default
The last point is key — Frappe always filters docstatus != 2 in list views unless you explicitly ask to see cancelled records. Your _handlers.select should do the same.
